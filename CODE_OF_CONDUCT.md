# Code of Conduct

This project is intended to be useful and approachable for operators, developers, and teams evaluating self-hosted collaboration tooling.

Participants are expected to:

- be respectful and constructive;
- focus criticism on code, behavior, documentation, and operational impact;
- avoid harassment, personal attacks, and discriminatory language;
- keep security reports private until maintainers have had time to respond.

Maintainers may edit, hide, or remove comments and issues that violate these expectations.
